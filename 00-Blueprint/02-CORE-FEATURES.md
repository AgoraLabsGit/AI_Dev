## 2. User Stories
> Break down the epics into specific user stories. This helps clarify the exact requirements for each feature.
>
> ### **User Authentication**
> - **As a** user, **I want to** be able to create a new account with my email and password **so that** I can access the system.
> - **As a** user, **I want to** be able to log in securely **so that** my business data is protected.
> - **As a** user, **I want to** be able to log out **so that** I can end my session.
>
> ### **Product & Menu Management**
> - **As an** admin, **I want to** be able to add new products with a name, price, and picture **so that** I can build my sales menu.
> - **As an** admin, **I want to** be able to edit existing products **so that** I can update pricing or details.
> - **As an** admin, **I want to** be able to temporarily disable a product **so that** I can mark it as "sold out."
>
> ### **Order Taking & Cart Management**
> - **As a** cashier, **I want to** be able to select products from a visual grid **so that** I can quickly add them to a customer's order.
> - **As a** cashier, **I want to** be able to view all items in the current cart **so that** I can confirm the order with the customer.
> - **As a** cashier, **I want to** be able to remove items from the cart **so that** I can correct mistakes.
>
> ### **Payment Processing**
> - **As a** cashier, **I want to** be able to initiate a credit card payment **so that** I can charge the customer.
> - **As a** cashier, **I want to** see a clear confirmation when a payment is successful **so that** I can complete the transaction.
>
> ### **Basic Transaction History**
> - **As an** admin, **I want to** be able to view a list of all transactions from the current day **so that** I can reconcile my sales.
> - **As an** admin, **I want to** be able to see the details of a specific transaction **so that** I can investigate any issues.
