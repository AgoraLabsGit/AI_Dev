# User Authentication - Overview

## 1. Page Purpose
- To allow users to securely sign in to the application.
- To provide a form for new users to register.
- To handle password recovery flows.

## 2. User Stories
- As a registered user, I want to log in with my email and password so that I can access my account.
- As a new user, I want to create an account so that I can use the application.

## 3. Acceptance Criteria
- [ ] Users can successfully log in with correct credentials.
- [ ] Users see an error message with incorrect credentials.
- [ ] New users can register and are automatically logged in.
